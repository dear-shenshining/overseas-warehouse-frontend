## 🎯 这个项目是什么？

想象一下，您有一个仓库管理系统，就像您在手机上用的APP一样。这个项目就是一个**网页版的仓库管理系统**，主要用来管理：

1. **海外物流管理** - 查看和跟踪从中国运到海外的货物
2. **日利润报表** - 查看每天赚了多少钱，用图表直观显示

这个系统是给电脑浏览器使用的，打开浏览器就能用，就像打开淘宝网一样简单。

## 📁 项目文件夹说明（用大白话说）

这个项目就像一套房子，不同的房间放不同的东西：

### 🏠 主要房间（文件夹）：

**`app/` 文件夹** - 就像房子的"客厅"
- 这里是整个网站的主页面
- `page.tsx` 是首页（就像您打开网站看到的第一页）
- `layout.tsx` 是整个网站的"装修风格"（比如标题、字体等）

**`components/` 文件夹** - 就像房子的"家具库"
- 这里存放所有可以重复使用的"零件"
- `overseas-logistics.tsx` - 海外物流页面的内容
- `daily-profit-report.tsx` - 利润报表页面的内容
- `ui/` 子文件夹里有很多小零件，比如按钮、输入框、卡片等

**`public/` 文件夹** - 就像房子的"图片相册"
- 这里存放网站用到的图片、图标等文件

**`lib/` 文件夹** - 就像房子的"工具箱"
- 这里存放一些常用的工具函数（帮助代码运行的辅助工具）

**`hooks/` 文件夹** - 就像房子的"智能开关"
- 这里存放一些可以让页面变得更智能的小程序

**`node_modules/` 文件夹** - ⚠️ **不要动这个文件夹！**
- 这是存放所有"零件包"的地方（就像乐高积木的零件盒）
- 这个文件夹是自动生成的，您不需要管它

**`package.json` 文件** - 就像项目的"购物清单"
- 这个文件记录了项目需要哪些"零件包"
- 也记录了可以执行哪些操作（比如启动、构建等）

## 🛠️ 这个项目用什么技术做的？

### 核心框架（就像房子的地基）：

**Next.js** - 这是一个制作网站的工具
- 就像您用Word写文档一样，Next.js是用来做网站的
- 它让网站运行更快，用户使用更流畅

**React** - 这是制作网页界面的工具
- 就像您用PPT做幻灯片一样，React是用来做网页界面的
- 它让网页可以"动起来"（比如点击按钮会有反应）

**TypeScript** - 这是让代码更安全的工具
- 就像给代码加上了"检查员"，避免出错

### 样式美化（就像房子的装修）：

**Tailwind CSS** - 用来美化网页的工具
- 就像给网页"化妆"，让网页看起来漂亮
- 可以设置颜色、字体、间距等

### 功能组件（就像房子的电器）：

**Radix UI** - 提供现成的按钮、输入框等小零件
- 就像买现成的家具，不用自己从头做

**Recharts** - 用来画图表的工具
- 就像用Excel画图表一样，可以画折线图、柱状图等

**Lucide React** - 提供各种图标
- 就像字体里的emoji，提供各种小图标

## 🚀 如何运行这个项目？（一步一步来）

### 第一步：确保您的电脑上安装了 Node.js

**Node.js是什么？**
- 就像您的电脑需要操作系统才能运行软件一样，运行这个项目需要 Node.js
- Node.js 让您的电脑能够理解和运行这个项目的代码

**如何检查是否安装了？**
1. 按 `Win + R` 键（Windows键 + R键）
2. 输入 `cmd` 然后按回车
3. 在黑色窗口里输入：`node --version`
4. 如果显示版本号（比如 v24.12.0），说明已经安装了
5. 如果没有显示，需要先安装 Node.js

**如何安装 Node.js？**
1. 打开浏览器，访问：https://nodejs.org/
2. 下载推荐版本（LTS版本）
3. 双击下载的文件，一路点击"下一步"完成安装
4. 安装完成后，重启电脑

### 第二步：安装项目依赖（就像安装软件需要的插件）

**依赖是什么？**
- 就像您要玩一个游戏，需要先安装这个游戏需要的所有插件一样
- 项目依赖就是项目运行需要的所有"零件包"

**如何安装？**
1. 打开命令提示符（cmd）
2. 用 `cd` 命令进入到项目文件夹：
   ```
   cd C:\Users\Administrator\Desktop\海外仓前端页面
   ```
3. 输入安装命令：
   ```
   npm install
   ```
4. 等待安装完成（可能需要几分钟）
5. 看到 `added XXX packages` 就说明安装成功了

### 第三步：启动开发服务器（就像打开网站）

**开发服务器是什么？**
- 就像您打开一个网站，需要在浏览器输入网址一样
- 开发服务器就是让您能在浏览器里看到这个项目的工具

**如何启动？**
1. 确保您在项目文件夹里（和第二步一样）
2. 输入启动命令：
   ```
   npm run dev
   ```
3. 等待几秒钟，您会看到类似这样的信息：
   ```
   ▲ Next.js 16.0.10
   - Local:        http://localhost:3000
   ```
4. 打开浏览器，输入：`http://localhost:3000`
5. 您就能看到网站了！🎉

**如何停止服务器？**
- 在命令提示符窗口按 `Ctrl + C`

## 📖 项目的两个主要功能

### 1️⃣ 海外物流管理页面

**这个页面做什么？**
- 显示所有海外物流订单的列表
- 可以搜索货运单号
- 显示统计数据：总运单数、运输中的订单、本周新增订单
- 显示每个订单的详细信息：
  - 货运单号
  - 订单状态（运输中、已送达、清关中）
  - 起运地和目的地
  - 发货日期和预计到达日期
- 可以导出数据

**如何查看？**
1. 启动项目后，默认就显示这个页面
2. 或者点击左侧菜单的"海外物流"按钮

### 2️⃣ 日利润报表页面

**这个页面做什么？**
- 显示关键数据卡片：
  - 总收入
  - 总利润
  - 利润率
  - 退款率
- 显示利润趋势折线图（可以看到利润和收入的变化）
- 显示退款分析柱状图（可以看到每天的退款情况）

**如何查看？**
1. 启动项目后，点击左侧菜单的"日利润报表"按钮

## 📝 常用命令说明

在项目文件夹里，您可以执行以下命令：

### `npm install`
- **作用**：安装项目需要的所有依赖（零件包）
- **什么时候用**：第一次运行项目，或者项目更新后
- **需要时间**：几分钟

### `npm run dev`
- **作用**：启动开发服务器，在浏览器查看网站
- **什么时候用**：想要查看和测试网站功能时
- **结果**：浏览器打开 http://localhost:3000 可以看到网站

### `npm run build`
- **作用**：把项目"打包"成可以发布到网上的版本
- **什么时候用**：想要把网站放到服务器上让其他人访问时
- **结果**：生成一个 `.next` 文件夹，里面是打包好的文件

### `npm start`
- **作用**：运行打包好的项目（需要先执行 `npm run build`）
- **什么时候用**：在生产环境（服务器上）运行网站时

### `npm run lint`
- **作用**：检查代码有没有错误或格式问题
- **什么时候用**：想要确保代码质量时

### 修改网页标题
- 打开 `app/layout.tsx` 文件
- 找到 `title: '金焱焱erp'` 这一行
- 改成您想要的标题

- 打开 `app/page.tsx` - 修改主页面
- 打开 `components/overseas-logistics.tsx` - 修改物流页面内容

- 打开 `app/globals.css` 文件

## ⚠️ 注意事项

1. **不要删除 `node_modules` 文件夹**
   - 这个文件夹很大，但它是自动生成的
   - 如果删除了，运行 `npm install` 可以重新生成

2. **不要删除 `package.json` 文件**
   - 这个文件记录了项目的重要信息

3. **修改代码后需要保存文件**
   - 如果开发服务器正在运行，保存后会自动刷新网页

4. **如果遇到错误**
   - 首先检查是否安装了 Node.js
   - 检查是否在正确的文件夹里运行命令
   - 尝试删除 `node_modules` 文件夹，然后重新运行 `npm install`

## 🤔 常见问题

### Q: 为什么我打开网页看不到内容？
A: 确保您已经执行了 `npm run dev` 命令，并且没有关闭命令提示符窗口

### Q: 修改代码后网页没有变化？
A: 检查是否保存了文件，开发服务器会自动检测变化并刷新

### Q: 如何添加新功能？
A: 需要修改相应的文件，比如添加新页面需要修改 `app/page.tsx` 或创建新文件

### Q: 项目里的数据是真实的吗？
A: 项目使用 Server Actions 直接连接数据库，数据从 MySQL 数据库的 `post_searchs` 表中获取
